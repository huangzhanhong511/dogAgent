# dogAgent 项目目录结构说明

## 顶层目录

```
dogAgent/
├── dogAgent/          # 主项目代码
├── helpMd/            # 技术文档（架构/记忆/知识库实现说明）
├── scripts/           # 运维脚本（定期维护等）
├── Makefile           # 统一命令入口（make serve / make test / make lint 等）
└── .gitignore
```

## dogAgent/ — 主项目

### agent/ — 核心 Agent 模块

这是整个系统的大脑，所有智能逻辑都在这里。

| 文件 | 职责 | 一句话说明 |
|------|------|-----------|
| `llm.py` | LLM 工厂 | **统一入口**，所有模块通过 `create_llm()` 获取 LLM 实例，支持 Qwen / OpenAI |
| `chat.py` | 对话引擎 | **粘合层**，串联检索 → 记忆 → LLM → tool calling → 持久化的完整链路 |
| `memory.py` | 记忆系统 | 5 张 SQLite 表（conversations, summaries, summary_edges, context_items, user_preferences），LCM 核心 |
| `compaction.py` | DAG 压缩引擎 | 当对话 token 超预算时，把旧消息压缩为层级摘要（leaf → condensed） |
| `memory_drilldown.py` | 无损回溯 | LLM 通过 memory_expand tool 展开摘要，逐层回溯到原始消息 |
| `session.py` | 会话管理 | 多会话创建/切换/归档/自动标题生成 |
| `background.py` | 后台任务 | 线程池异步执行偏好提取/压缩/标题生成，不阻塞用户 |
| `query_rewrite.py` | 查询重写 | 结合上下文把"它怎么了"补全为"旺旺的耳朵怎么了" |
| `retriever.py` | 检索器 | **两层 LLM 索引检索**（顶层 → 分类 → 文章）+ 规则匹配 fallback |
| `build_wiki.py` | Wiki 生成 | 把 knowledge/ 的多源文章通过 LLM 合并为结构化中文 Wiki |
| `build_index.py` | 索引生成 | 从 wiki/*.md 提取关键词/章节/摘要，生成两层索引（index.md + 分类 index.md） |
| `generate_topics.py` | LLM 补缺 | 对比 target_topics.json 找缺失主题，用 LLM 生成文章填充 knowledge/ |
| `wiki_lint.py` | 知识库维护 | 审计报告/清理无关文章/LLM 交叉引用/修复降级文章 |
| `sync_to_obsidian.py` | Obsidian 同步 | 把 wiki/ 同步到 Obsidian Vault（可选） |

### api/ — REST API 服务

| 文件 | 职责 |
|------|------|
| `server.py` | FastAPI 应用主文件，定义所有端点（auth/chat/sessions/preferences），初始化全局组件 |
| `auth.py` | JWT 认证服务（注册/登录/token 校验） |
| `models.py` | Pydantic 请求/响应模型定义 |

### web/ — 前端 Web UI

| 文件 | 职责 |
|------|------|
| `index.html` | 单文件 SPA（HTML + CSS + JS），包含登录/注册、对话、会话管理、偏好面板、摘要卡片 |

```
├── index.json            # 完整检索索引（关键词/章节/标签/摘要）
├── 01-品种百科/
│   ├── index.md          # 分类级索引（每篇文章一行摘要）
│   ├── 迷你雪纳瑞.md
│   ├── 标准雪纳瑞.md
│   └── 巨型雪纳瑞.md
├── 02-饮食营养/          # 喂食指南、禁忌食物、胰腺炎饮食等
├── 03-健康医疗/          # 胰腺炎、膀胱结石、耳部感染等 47 篇
├── 04-美容护理/          # 手剥技术、造型指南、耳部清洁等
├── 05-训练与行为/        # 吠叫控制、社会化、服从训练等
├── 06-日常饲养/          # 运动量、新犬到家、季节性护理等
├── 07-繁殖与幼犬/        # 犬舍选择、遗传筛查、幼犬疫苗等
```

### knowledge/ — 原始知识源（Wiki 的输入）

```
knowledge/
├── 00-索引/              # MOC 总览
├── 01-品种百科/          # PetMD 爬取 + LLM 生成的品种文章
├── 02-饮食营养/          # LLM 生成的饮食文章
├── 03-健康医疗/          # PetMD 爬取的 500+ 篇英文疾病文章 + LLM 中文文章
├── 04-美容护理/          # LLM 生成
├── 05-训练与行为/
├── 06-日常饲养/
├── 07-繁殖与幼犬/
├── 08-法规与养犬常识/
├── 09-参考资料/
└── 模板/                 # Wiki 模板（疾病模板、食谱模板、药物模板、案例模板）
```

**knowledge/ 和 wiki/ 的区别**：
- `knowledge/` = 原始素材（爬来的英文 + LLM 生成的中文），可能有重复和多语言
- `wiki/` = 最终产物（build_wiki.py 合并去重后的结构化中文 Wiki），这是检索器读取的内容

### crawlers/ — 数据爬虫

```
crawlers/
├── config/
│   ├── sources.json        # 爬虫源配置（URL、关键词、可靠度）
│   └── target_topics.json  # 52 个目标主题清单（驱动 LLM 补缺）
├── scripts/
│   ├── base_crawler.py     # 爬虫基类（重试、去重、HTML→Markdown）
│   ├── crawl_petmd.py      # PetMD 爬虫（疾病百科、营养、护理）
│   ├── crawl_akc.py        # AKC 爬虫（品种标准、训练）
│   ├── crawl_vca.py        # VCA 爬虫（兽医院指南）
│   ├── crawl_aspca.py      # ASPCA 爬虫（有毒食物）
│   ├── crawl_boqii.py      # 波奇网爬虫（中文资源）
│   ├── crawl_reddit.py     # Reddit 爬虫（社区经验）
│   ├── run_all.py          # 批量运行所有爬虫
│   └── clean_and_organize.py  # 清洗 + 按分类整理到 knowledge/
├── raw_data/               # 爬取的原始 HTML（不上传 GitHub）
└── cleaned_data/           # 转换后的 Markdown（不上传 GitHub）
```

### data/ — 运行时数据（不上传 GitHub）

```
data/
├── memory.db    # SQLite 数据库（对话、摘要、偏好、会话、用户）
```

### conflicts/ — 冲突报告

build_wiki.py 合并多源文章时，如果发现信息不一致（如 PetMD 和 VCA 对某个剂量说法不同），会记录在 `冲突报告.md` 中，供人工审核。

## 数据流向总览

```
爬虫 (crawlers/) → 原始数据 (raw_data/) → 清洗 (cleaned_data/)
                                                    ↓
                                            knowledge/ (知识源)
                                                    ↓
        LLM 补缺 (generate_topics.py) ──────→ knowledge/
                                                    ↓
                                build_wiki.py (合并+翻译+去重)
                                                    ↓
                                              wiki/ (最终 Wiki)
                                                    ↓
                                build_index.py (两层索引)
                                                    ↓
                              index.md + {分类}/index.md + index.json
                                                    ↓
                        retriever.py (LLM 读索引 → 选文章 → 加载内容)
                                                    ↓
                              chat.py (检索 + 记忆 + LLM → 回答用户)
```
